# QwenScan

Система автоматического распознавания и извлечения структурированных данных из сканированных PDF-документов на основе мультимодальных VLM-моделей (Qwen3-VL).

---

## Возможности

- **Пакетная обработка** — загрузка тысяч PDF одновременно, параллельная обработка
- **Авто-чанкинг** — документы с большим количеством страниц автоматически разбиваются на части и обрабатываются независимо
- **Мультимодельность** — переключение между моделями (8B быстрая / 35B качество) через UI без перезапуска сервера
- **20 шаблонов** для российских бухгалтерских документов с умной пост-обработкой (ИНН/КПП, адреса, суммы)
- **Авто-определение шаблона** — система сама распознаёт тип документа и выбирает нужный промпт
- **Чат с документами** — задавать вопросы по загруженным PDF прямо в интерфейсе
- **Экспорт в Excel** — структурированная выгрузка с гиперссылками на исходные файлы
- **SSE-прогресс** — обновление статуса в реальном времени, ETA, скорость обработки
- **Возобновление** — прерванный батч продолжается с того места, где остановился

## Архитектура

```
┌──────────────────────────────────────────────┐
│            Веб-интерфейс (порт 8081)         │
│  Загрузка PDF · Прогресс SSE · Результаты   │
└───────────────────┬──────────────────────────┘
                    │ HTTP / SSE
┌───────────────────▼──────────────────────────┐
│             FastAPI (main.py)                │
│  /api/upload  /api/batch/*/progress          │
│  /api/model/* /api/chat  /api/batch/*/excel  │
└──────┬──────────┬──────────────┬─────────────┘
       │          │              │
       ▼          ▼              ▼
  batch_       pipeline.py  model_manager.py
  processor.py (VLM-пайплайн) (запуск/выгрузка GPU)
  (очередь PDF)    │              │
       │           ▼              ▼
       │      vlm_client.py  start_vllm.sh /
       │      (HTTP → vLLM)  start_vllm_35b.sh
       ▼           │
  pdf_utils.py     │
  (PDF → PNG)      ▼
              ┌──────────────┐
              │  vLLM :8001  │
              │  Qwen3-VL    │
              │  (8B / 35B)  │
              └──────────────┘
```

Два независимых процесса:
- **FastAPI** (порт 8081) — API, координация, хранение результатов в SQLite
- **vLLM** (порт 8001) — inference-сервер, OpenAI-совместимый API

## Стек технологий

| Компонент | Технология |
|---|---|
| Модели | Qwen3-VL-8B-Instruct-AWQ-8bit, Qwen3.5-35B-A3B-GPTQ-Int4 |

| Realtime | sse-starlette (Server-Sent Events) |

## Установка

### Требования

- Python 3.12+
- NVIDIA GPU: 24+ GB VRAM для 8B модели, 48+ GB для 35B
- CUDA 12.x
- ~12 GB диска для 8B модели, ~20 GB для 35B

### Зависимости

```bash
git clone https://github.com/Starlight19973/QwenScan.git
cd QwenScan
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

### Скачать модель

```bash
# 8B модель (рекомендуется для начала)
huggingface-cli download cyankiwi/Qwen3-VL-8B-Instruct-AWQ-8bit \
  --local-dir models/Qwen3-VL-8B-Instruct-AWQ-8bit

# 35B модель (лучшее качество, нужно 48 GB VRAM)
huggingface-cli download Qwen/Qwen3.5-35B-A3B-GPTQ-Int4 \
  --local-dir models/Qwen3.5-35B-A3B-GPTQ-Int4
```

## Запуск

```bash
# 1. Запустить inference-сервер (vLLM)
bash start_vllm.sh          # 8B модель
# или
bash start_vllm_35b.sh      # 35B модель

# 2. Запустить веб-приложение
bash start_app.sh
```

Веб-интерфейс: `http://localhost:8081`

> Модель можно также загружать и выгружать прямо из веб-интерфейса — кнопка в шапке страницы.

## Конфигурация

Основные параметры в `app/config.py`:

| Параметр | По умолчанию | Описание |
|---|---|---|
| `MAX_CONCURRENT_REQUESTS` | `2` | Параллельных запросов к VLM (сохраняется в `settings.json`) |
| `MAX_DOCUMENT_PAGES` | `20` | Макс. страниц без чанкинга |
| `EXTRACTION_CHUNK_SIZE` | `12` | Страниц в одном чанке при авто-разбивке |
| `PDF_DPI` | `260` | DPI конвертации PDF → PNG |
| `MAX_MODEL_LEN` | `32768` | Контекстное окно vLLM |
| `EXTRACTION_MAX_TOKENS` | `6144` | Макс. токенов ответа модели |
| `REQUEST_TIMEOUT` | `600.0` | Таймаут запроса к VLM (сек) |

Параллельность можно изменять через API без перезапуска:
```
POST /api/concurrency  {"value": 4}
```

## Шаблоны документов

Система поддерживает 20 шаблонов российских документов с автоматическим определением типа:

| Шаблон | Описание |
|---|---|
| `universal` | Универсальный (любой документ) |
| `schet_faktura` | Счёт-фактура |
| `upd` | УПД (Универсальный передаточный документ) |
| `akt` | Акт выполненных работ |
| `torg12` | Товарная накладная ТОРГ-12 |
| `dogovor` | Договор |
| `schet_na_oplatu` | Счёт на оплату |
| `platezhnoe` | Платёжное поручение |
| `pasport` | Паспорт гражданина РФ |
| `doverennost` | Доверенность |
| `deklaratsiya` | Декларация |
| `transportnaya` | Транспортная накладная |
| `tekst` | Произвольный текст |
| `dokumenty_sezar` | Специализированный шаблон (Сезар) |
| ... | и ещё 6 шаблонов |

### Авто-режим

При включённом авто-режиме система:
1. Пробует определить шаблон по имени файла
2. Анализирует первые страницы документа VLM-моделью
3. Выбирает наиболее подходящий шаблон
4. Применяет соответствующий промпт

## Авто-чанкинг больших документов

Документы с количеством страниц больше `EXTRACTION_CHUNK_SIZE` (12 по умолчанию) автоматически разбиваются на части:

```
документ.pdf (100 стр.) →
  документ [стр. 1–12].pdf
  документ [стр. 13–24].pdf
  ...
  документ [стр. 97–100].pdf
```

- Исходный файл скрывается из UI (статус `split`)
- Чанки обрабатываются параллельно
- Прогресс и результаты отображаются отдельно по каждому чанку
- При возобновлении прерванного батча чанки пересчитываются из исходного PDF

## API

Полное описание: [`docs/API_QWEN.md`](docs/API_QWEN.md)

### Основные эндпоинты

```
POST /api/upload                      — загрузить PDF-файлы, получить batch_id
GET  /api/batch/{id}/progress         — SSE-поток прогресса
GET  /api/batch/{id}/results          — извлечённые данные JSON
POST /api/batch/{id}/generate-excel   — сгенерировать Excel
GET  /api/batch/{id}/excel            — скачать Excel-файл
GET  /api/batches                     — список всех батчей

POST /api/model/load                  — загрузить модель в GPU
POST /api/model/unload                — выгрузить модель из GPU
GET  /api/model/status                — статус модели

POST /api/chat                        — чат с документами
POST /api/concurrency                 — изменить параллельность
```

## Пост-обработка данных

После извлечения данных моделью применяется автоматическая коррекция:

- **ИНН/КПП**: разделение объединённых значений (`7723022111/772201001`), удаление КПП у ИП
- **Адреса**: очистка от банковских реквизитов, телефонов, e-mail
- **Номера документов**: исправление `О` → `0`, удаление лишних пробелов
- **Суммы**: корректное слияние многостраничных документов (ИТОГО всегда с последней страницы)
- **Валидация ИНН**: проверка по контрольным суммам (алгоритм ФНС)
- **Слияние страниц**: поля с нескольких страниц объединяются, товарные позиции перенумеровываются

## Требования к серверу

| Конфигурация | GPU | Модель |
|---|---|---|
| Минимальная | 24 GB VRAM | Qwen3-VL-8B-AWQ |
| Рекомендуемая | 48 GB VRAM | 8B + возможность запуска 35B |
| Максимальная | 48 GB VRAM | Qwen3.5-35B-A3B-GPTQ-Int4 |

Скорость обработки на RTX 4090 (48 GB):
- **8B модель**: ~5–10 сек/страница при параллельности 4–8
- **35B модель**: ~15–30 сек/страница
